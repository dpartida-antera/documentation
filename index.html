PO Shipping Address Guide

How the system decides where every purchase order ships — written for QA and Support. Start with the mental model, then use the interactive tool to confirm any case in seconds.

Intended behavior reference · Standard & Store orders

01How shipping works, in one minute

Every order is broken into one or more Purchase Orders (POs). Each PO is a document telling one party to send goods somewhere. The system answers a single question for each PO: “What address goes in the Ship To block?”

The whole system is just three inputs feeding one decision.
1) Which document is this? (Blank PO, Decorator PO, or Supplier Decorated PO)  →  2) Which flags are set on the order/product?  →  3) The system walks a decision tree and picks one of seven possible destination addresses.

The goal is to route product through the right hands: a blank garment may need to reach a decorator before it reaches the customer; a finished item may go straight to the buyer; sometimes everything routes back to your own warehouse first. The flags below are how a CSR expresses that intent, and this guide is how you confirm the system honored it.

02The three PO documents

A single order can generate more than one of these. They each follow their own decision tree because they sit at different points in the production chain.

Blank PO

Undecorated goods from a supplier

Orders blank product from the blank supplier. If the item is decorated, the blank usually has to reach a decorator before going anywhere else.

Decorator PO

Sends work to a decorator

Goes to the vendor doing the decoration (embroidery, print, etc.) when decoration is done by a different vendor than the blank supplier.

Supplier Decorated PO

Supplier also decorates

Used when the product and its decoration come from the same supplier. Follows the same address logic as a Decorator PO.

Multi-decoration chains: when an item carries decorations from several vendors, POs hand off in sequence — Supplier → Decorator 1 → Decorator 2 → … → final destination. Only the final PO in the chain ships to the customer (or Alternate address); every earlier PO ships to the next decorator.

04The seven destination addresses

Every decision ends at exactly one of these. Knowing the source of each is what lets you verify the address block field-by-field.

DestinationSource of the data
Order Shipping AddressThe customer address on the Order (the default end destination).
Alternate Ship To AddressThe override address typed into the Alternate Ship To fields.
First Decorator AddressThe shipping address of the first decorator vendor.
Second & Beyond DecoratorThe shipping address of the next decorator in a multi-vendor chain.
Destination WarehouseThe configured Destination Warehouse address.
Corporate Identity AddressThe Corporate Identity shipping address on the account.
Partner AddressYour own Partner Company address (system default for Ship to Us).
A Supplier PO in a standard “ship to decorator” flow ships from the Vendor — its address block is sourced from the Vendor record, not from the list above.

05Priority rules (conflict resolution)

When more than one flag is set, the system resolves conflicts in a fixed order. This single rule explains most “why did it ship there?” questions.

1 · Highest

Alternate Ship To

If an Alternate address is set, it wins over Ship-to-Us and default routing on the final PO.

2

Ship to Us settings

If no Alternate address, Ship-to-Us routing (warehouse → corporate identity → partner) applies.

3 · Default

Vendor / Order

With no overrides, the Supplier PO ships from Vendor and the Decorator PO ships to the Order address.

Within “Ship to Us,” there is a sub-priority: Destination Warehouse → then Corporate Identity → then Partner Address. The system takes the first one that is configured.

    11Glossary

    PO (Purchase Order)A document instructing one party to supply/decorate/ship goods. One order can spawn several.
    Blank / Blank SupplierUndecorated product and the vendor that provides it.
    DecoratorThe vendor that applies artwork (embroidery, screen print, etc.).
    Final / Final Decorator POThe last PO in a decoration chain — the one that ships to the customer or Alternate address.
    Ship to UsRouting goods back to your own location (warehouse → corporate identity → partner) instead of onward.
    Corporate IdentityA branded account-specific shipping location used for Ship-to-Us when no warehouse is set.
    Partner AddressYour own company's default address — the final fallback for Ship-to-Us.
    Alternate Ship ToA one-off override address entered on the order; highest routing priority.
